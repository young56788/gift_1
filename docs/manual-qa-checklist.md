# 手工验收清单

建议每次走查前复制一份 [manual-qa-report-template.md](/home/youngsun/projects/gift_1/docs/manual-qa-report-template.md) 记录结果、复现步骤和截图路径。

## 基础验证
- 运行 `npm run build`，确保 TypeScript 检查和 Vite 构建通过。
- 运行 `npm run dev`，确认首屏可以进入入口页，且浏览器控制台没有明显报错。
- 在入口页点击“进入地图”，确认 Phaser 舞台正常加载地图。

## 主线闭环
### 路径 1：先钓鱼再卡坦
- 从地图进入钓鱼。
- 完成钓鱼，确认返回地图，且能继续移动与交互。
- 确认金币显示与本局结算一致（不出现黑屏或卡死）。
- 从地图进入卡坦，完成整局。
- 返回地图后确认广场状态变为“已解锁”。
- 进入宴会，确认剧情能顺序播放到结尾。

### 路径 2：先卡坦再钓鱼
- 从地图直接进入卡坦并完成。
- 返回地图后确认广场仍未解锁。
- 再进入钓鱼并完成。
- 返回地图后确认广场立即解锁。

### 路径 3：中途退出
- 进入钓鱼后直接退出，确认回到地图且可再次进入钓鱼。
- 进入卡坦后直接返回地图，确认小岛未标记完成。

## 卡坦专项
- 进入卡坦后，确认默认操作依赖棋盘高亮点位，右侧按钮默认收起。
- 悬停可点击道路或据点，确认左侧说明面板会更新动作标题、描述和预览。
- 点击热点后，确认 React 正常结算，Phaser 会显示落子反馈。
- 验证骰子、资源飞字、强盗移动、偷牌提示、最长道路、最大骑士提示都会触发。
- 验证发展卡库存会更新，且“抽到/打出”会在棋盘上出现演出反馈。

## 宴会专项
- 触发晚会后确认主角可见、NPC 祝福台词顺序正常、烟花演出正常。
- 礼物交互后可见纸条金额与暗号，水库宝箱可见“玉石挂坠一个”。
- 晚会结束返回地图后，场景与交互保持可用。

## 已知关注点
- `phaser-runtime` 仍是大 chunk，但已经不在首屏主包内，这属于当前可接受状态。
- 当前没有自动化测试，发版前至少完整走一遍“路径 1”和“路径 2”。
